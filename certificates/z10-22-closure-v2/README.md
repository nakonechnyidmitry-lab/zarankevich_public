# z(10,22;3,3) = 110 — независимый машинно-проверяемый UNSAT-сертификат (v2, реплеемый)

**Что доказано.** z(10,22;3,3) = 110: нижняя граница — конструкция-свидетель
(проверяется `check_witness.py`), верхняя — UNSAT при 111 единицах.

**Почему это не тривиально.** Верхняя граница 110 НЕ выводится deletion-арифметикой
из опубликованных конкурентских значений (замыкание упирается в 111). Наш UNSAT@111 —
единственный известный машинно-проверяемый путь к этой границе. Значение z(10,22)=110 первым явно вписал dfield (артефактов не публиковал); наш вклад — независимый реплеемый машинный сертификат обеих границ.

**Реплеемость.** Пакет z10-22-k111-v2 (на архивном носителе / Zenodo) содержит
самодостаточные ПАРЫ `cube_X.cnf.xz` + `cube_X.lrat.xz` для каждого листа
(база + march-хвост, если был). Проверка каждого листа:
```
unxz cube_X.cnf.xz cube_X.lrat.xz
lrat-check cube_X.cnf cube_X.lrat   # => "s VERIFIED"
```
Все листы UNSAT => z <= 110. Хэши — в SHA256SUMS.replayable. `verify.sh` — с нуля
(детерминированный ре-эмит tan-кубов + пере-решение).
Пруфов+CNF: 55 пар.
